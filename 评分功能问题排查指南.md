### 2. 检查小程序端

1. 打开小程序开发者工具
2. 确保已登录（检查 `app.globalData.isLoggedIn`）
3. 进入美食详情页
4. 选择评分（1-5星）
5. 输入评论内容
6. 点击提交
7. 观察控制台输出

#### 查看小程序控制台日志：
```

# 失败的情况可能会看到：

```

### 3. 检查数据库

连接数据库，执行以下查询：

```sql
-- 查看最新的评论记录
SELECT * FROM comments ORDER BY created_at DESC LIMIT 10;

-- 查看某个美食的评论和评分
SELECT * FROM comments WHERE food_id = 'YOUR_FOOD_ID' ORDER BY created_at DESC;

-- 统计某个美食的评分信息
SELECT
    COUNT(*) as total_comments,
    COUNT(CASE WHEN rating IS NOT NULL AND rating > 0 THEN 1 END) as rated_comments,
    AVG(CASE WHEN rating IS NOT NULL AND rating > 0 THEN rating END) as avg_rating
FROM comments
WHERE food_id = 'YOUR_FOOD_ID';
```

### 5. 测试网络请求

2. 找到 `POST /api/mini/foods/{id}/comments` 请求
3. 查看请求头中的 Authorization 字段

5. 查看响应内容

## 常见问题和解决方案

### 问题1：提示"请先登录"
**原因**：用户未登录或 token 过期
**解决方案**：
1. 检查小程序登录逻辑
2. 确认 token 是否正确存储在本地存储中
3. 检查后端的 token 验证逻辑

### 问题2：提示"评分必须在1-5之间"
**原因**：评分值不合法
**解决方案**：
1. 检查前端评分组件的返回值
2. 确保评分是数字类型且在 1-5 范围内

### 问题3：提示"评论内容不能为空"
**原因**：评论内容为空或只包含空格
**解决方案**：
1. 检查前端输入框的值
2. 确保内容经过 trim() 处理后不为空

### 问题4：提示"提交失败"但没有详细错误
**原因**：可能是数据库连接问题或插入失败
**解决方案**：
1. 检查数据库连接配置
2. 查看 MySQL 日志
3. 检查 comments 表结构是否正确

### 问题5：首页评分没有更新
**原因**：前端缓存了旧数据
**解决方案**：
1. 下拉刷新首页
2. 重新进入首页
3. 检查首页的 `onShow` 方法是否正确调用

## 数据库表结构确认

确保 `comments` 表结构正确：

```sql
CREATE TABLE IF NOT EXISTS `comments` (
    `id` VARCHAR(64) PRIMARY KEY,
    `food_id` VARCHAR(64) NOT NULL,
    `user_id` VARCHAR(64) NOT NULL,
    `content` TEXT NOT NULL,
    `rating` INT DEFAULT 0,
    `images` VARCHAR(1000),
    `status` VARCHAR(20) DEFAULT 'pending',
    `created_at` DATETIME NOT NULL,
    INDEX idx_food_id (`food_id`),
    INDEX idx_user_id (`user_id`),
    INDEX idx_created_at (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
