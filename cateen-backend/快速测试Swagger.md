## 📍 第三步：测试登录接口

### 1. 找到"1. 认证模块"
- 点击展开
- 找到 **POST /api/auth/login** 接口

### 2. 点击"调试"按钮

### 3. 输入测试数据

在请求体中输入：
```json
{
  "username": "admin",
  "password": "admin123"
}
```

### 4. 点击"发送"按钮

### 5. 查看响应

成功响应示例：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOiIxIiwidXNlcm5hbWUiOiJhZG1pbiIsInJvbGUiOiJhZG1pbiIsInN1YiI6ImFkbWluIiwiaWF0IjoxNjk2MzI5NjAwLCJleHAiOjE2OTY0MTYwMDB9.xxx",
    "user": {
      "id": "1",
      "username": "admin",
      "phone": "13800138000",
      "role": "admin",
      "status": "active",
      "createdAt": "2025-09-22T10:00:00"
    }
  }
}
```

### 6. 复制Token

从响应中复制 `data.token` 的值（整个长字符串）

---

## 📍 第四步：配置全局Token

### 方式一：使用Authorize按钮（推荐）⭐

1. **找到页面右上角的🔒图标或"Authorize"按钮**
2. **点击它**
3. **在弹出的输入框中输入**：
   ```
   Bearer eyJhbGciOiJIUzI1NiJ9.你的token...
   ```
   ⚠️ 注意：
   - 必须以 `Bearer ` 开头（Bearer后面有一个空格）
   - 然后跟上你刚才复制的完整token

4. **点击"Authorize"或"授权"按钮**
5. **点击"Close"关闭窗口**

✅ 现在所有需要认证的接口都会自动带上这个Token！

## 📍 常见问题

### ❓ 为什么接口返回401错误？

**原因**：没有配置Token或Token格式错误

**解决**：
1. 检查是否点击了Authorize按钮配置Token
2. 检查Token格式：必须是 `Bearer {token}`
3. 重新登录获取新Token

### ❓ Token在哪里配置？

**位置**：页面右上角的🔒图标或"Authorize"按钮

### ❓ 如何刷新Token？

1. Token有效期24小时
2. 过期后重新登录
3. 重新点击Authorize配置新Token

### ❓ 测试账户是什么？

**管理员账户**：
- 用户名：admin
- 密码：admin123

**普通用户**：
- 用户名：testuser
- 密码：123456
