# 评分关注点功能实现方案

## 一、数据库设计

### 方案选择：方案B（独立关注点表）+ 方案C（NLP分析）

**理由：**
- 数据结构清晰，易于查询和统计
- 支持多维度评分
- 可扩展性强，未来可以添加更多关注点类型
- 可以结合NLP进行智能分析

### 1.1 创建评论关注点表

```sql
CREATE TABLE IF NOT EXISTS `comment_preferences` (
    `id` VARCHAR(50) NOT NULL COMMENT '主键ID',
    `comment_id` VARCHAR(50) NOT NULL COMMENT '评论ID',
    `preference_type` ENUM('taste', 'price', 'environment', 'service', 'other') NOT NULL COMMENT '关注点类型',
    `score` INT NOT NULL COMMENT '该关注点的评分(1-5)',
    `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_comment_type` (`comment_id`, `preference_type`),
    KEY `idx_comment` (`comment_id`),
    KEY `idx_type` (`preference_type`),
    KEY `idx_score` (`score`),
    FOREIGN KEY (`comment_id`) REFERENCES `comments`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='评论关注点表';
```

### 2.2 Mapper接口

```java
@Mapper
public interface CommentPreferenceMapper extends BaseMapper<CommentPreference> {
    
    /**
     * 统计各关注点的平均评分和占比
     */

    /**
     * 根据食物ID统计关注点
     */
    List<Map<String, Object>> statisticsPreferencesByFood(@Param("foodId") String foodId);
    
    /**
     * 根据分类统计关注点
     */

}
```

### 2.3 Mapper XML

```xml
<!-- 统计各关注点的平均评分和占比 -->
<select id="statisticsPreferences" resultType="java.util.HashMap">
    SELECT
        preference_type AS preference,
        ROUND(AVG(score), 2) AS avg_score,
        COUNT(*) AS count,
        ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM comment_preferences), 2) AS percentage,
        CASE preference_type
            WHEN 'taste' THEN '#3b82f6'
            WHEN 'price' THEN '#10b981'
            WHEN 'environment' THEN '#f59e0b'
            WHEN 'service' THEN '#8b5cf6'
            ELSE '#6b7280'
        END AS color
    FROM comment_preferences
    GROUP BY preference_type
    ORDER BY count DESC
</select>

<!-- 根据食物ID统计关注点 -->
<select id="statisticsPreferencesByFood" resultType="java.util.HashMap">
    SELECT
        cp.preference_type AS preference,
        ROUND(AVG(cp.score), 2) AS avg_score,
        COUNT(*) AS count
    FROM comment_preferences cp
    INNER JOIN comments c ON cp.comment_id = c.id
    WHERE c.food_id = #{foodId}
    GROUP BY cp.preference_type
    ORDER BY count DESC
</select>
```

### 2.5 Service实现

```java
@Service
@RequiredArgsConstructor
public class CommentPreferenceServiceImpl implements CommentPreferenceService {
    
    private final CommentPreferenceMapper preferenceMapper;
    
    @Override
    public void createPreferences(String commentId, Map<String, Integer> preferences) {
        preferences.forEach((type, score) -> {
            if (score > 0) {
                CommentPreference preference = new CommentPreference();
                preference.setId(IdUtil.getSnowflakeNextIdStr());
                preference.setCommentId(commentId);
                preference.setPreferenceType(type);
                preference.setScore(score);
                preferenceMapper.insert(preference);
            }
        });
    }
    
    @Override
    public List<Map<String, Object>> getRatingPreferences() {
        List<Map<String, Object>> result = preferenceMapper.statisticsPreferences();
        
        // 如果没有数据，返回模拟数据
        if (result.isEmpty()) {
            return generateMockPreferences();
        }
        
        // 翻译为中文
        result.forEach(item -> {
            String preference = (String) item.get("preference");
            item.put("preference", translatePreference(preference));
        });
        
        return result;
    }
    
    @Override
    public Map<String, Integer> analyzeCommentContent(String content) {
        Map<String, Integer> preferences = new HashMap<>();
        
        // 简单的关键词匹配（实际可以使用更复杂的NLP算法）
        if (content.contains("好吃") || content.contains("美味") || content.contains("口味")) {
            preferences.put("taste", 5);
        }
        if (content.contains("实惠") || content.contains("便宜") || content.contains("性价比")) {
            preferences.put("price", 5);
        }
        if (content.contains("环境") || content.contains("装修") || content.contains("干净")) {
            preferences.put("environment", 5);
        }
        if (content.contains("服务") || content.contains("态度") || content.contains("热情")) {
            preferences.put("service", 5);
        }
        
        return preferences;
    }
    
    private String translatePreference(String preference) {
        Map<String, String> translations = Map.of(
            "taste", "口味",
            "price", "价格",
            "environment", "环境",
            "service", "服务",
            "other", "其他"
        );
        return translations.getOrDefault(preference, preference);
    }

    }
}
```

## 四、实施步骤

1. **阶段1：基础功能**
   - 创建数据库表
   - 实现基本的关注点CRUD
   - 修改评论API支持关注点数据

2. **阶段2：统计分析**
   - 实现关注点统计查询
   - 前端展示统计图表
   - 支持按分类、食物维度查看

3. **阶段3：智能分析**
   - 实现NLP关键词分析
   - 自动从评论内容提取关注点
   - 优化分析算法

4. **阶段4：高级功能**
   - 关注点趋势分析
   - 个性化推荐
   - 竞品对比分析

## 五、优势

1. **数据驱动**：基于真实用户反馈
2. **多维分析**：不仅看总分，还看各维度评分
3. **智能化**：自动分析评论内容
4. **可扩展**：易于添加新的关注点类型
5. **业务价值**：帮助商家了解用户真实需求

## 六、注意事项

1. **数据质量**：确保用户评分的真实性
2. **性能优化**：大数据量时需要优化查询
3. **隐私保护**：不泄露用户个人偏好
4. **定期更新**：关键词库需要定期维护
