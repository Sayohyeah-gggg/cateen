# 数据库表清理建议

## 可以删除的表

根据代码分析，以下表在当前系统中**没有被使用**或**数据为空且功能已删除**，可以安全删除：

### 1. preference_keywords (关注点关键词表)
- **状态**: 32 条数据
- **使用情况**: ❌ 代码中完全没有使用
- **建议**: **可以删除**
- **原因**: 没有任何 Java 类、Mapper 或 Service 引用此表

### 2. visit_stats (访问统计表)
- **状态**: 30 条数据（可能是空数据或测试数据）
- **使用情况**: ⚠️ 有代码但前端功能已删除
- **建议**: **可以删除**
- **原因**: 
  - 前端访问统计页面已删除
  - 管理端不再使用此功能
  - 小程序端也不需要此数据

### 3. device_stats (设备统计表)
- **状态**: 3 条数据（可能是空数据或测试数据）
- **使用情况**: ⚠️ 有代码但前端功能已删除
- **建议**: **可以删除**
- **原因**: 
  - 前端设备统计功能已删除
  - 管理端不再使用此功能

### 4. source_stats (来源统计表)
- **状态**: 5 条数据（可能是空数据或测试数据）
- **使用情况**: ⚠️ 有代码但前端功能已删除
- **建议**: **可以删除**
- **原因**: 
  - 前端来源统计功能已删除
  - 管理端不再使用此功能

## 删除 SQL 语句

如果确认要删除上述表，可以执行以下 SQL：

```sql
-- 删除未使用的表
DROP TABLE IF EXISTS preference_keywords;
DROP TABLE IF EXISTS visit_stats;
DROP TABLE IF EXISTS device_stats;
DROP TABLE IF EXISTS source_stats;
```

## 总结

- **可删除表**: 4 个（preference_keywords, visit_stats, device_stats, source_stats）
- **保留表**: 12 个（所有核心业务和功能表）
- **删除后收益**: 
  - 减少数据库维护成本
  - 简化代码结构
  - 提高系统稳定性
  - 避免空表导致的功能问题

## 注意事项

1. 删除前请先备份数据库
2. 确认生产环境中这些表确实没有被使用
3. 删除表后需要同步删除后端代码
4. 建议在测试环境先验证删除操作
