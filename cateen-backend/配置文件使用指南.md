# Spring Boot 配置文件使用指南

## 📁 配置文件说明

项目有3个配置文件：

```
application.yml          # 主配置（公共配置）
application-dev.yml      # 开发环境配置（本地测试用）⭐
application-prod.yml     # 生产环境配置（部署服务器用）
```

---

## 🎯 本地测试步骤

### 第1步：确认主配置文件激活了dev环境

打开 `application.yml`，确认有这行配置：
```yaml
spring:
  profiles:
    active: dev  # 👈 使用dev环境
```

✅ **这个配置已经设置好了，不需要修改！**

---

### 第2步：修改开发环境配置

打开 `application-dev.yml`，根据你的本地环境修改：

#### 2.1 MySQL数据库配置（必须修改）

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/cateen?...
    username: root
    password: 666Shiyuhang  # 👈 改成你的MySQL密码
```

**重要提示：**
- 确保MySQL已启动
- 确保数据库 `cateen` 已创建
- 确保已执行 `schema.sql` 和 `data.sql`

#### 2.2 Redis配置（可选）

如果你本地没有安装Redis，可以：
- **方案1**：注释掉Redis相关代码（如果暂时不用）
- **方案2**：安装Redis并启动

```yaml
spring:
  redis:
    host: localhost
    port: 6379
    password:  # 如果有密码就填写，没有就留空
```

---

### 第3步：启动项目

有3种启动方式，**任选一种**：

#### 方式1：IDEA运行（推荐）⭐

1. 右键点击 `CateenApplication.java`
2. 选择 `Run 'CateenApplication'`
3. 控制台会显示使用了dev环境：
   ```
   The following 1 profile is active: "dev"
   ```

#### 方式2：Maven命令运行

```bash
mvn spring-boot:run
```

#### 方式3：打包后运行

```bash
# 打包
mvn clean package -DskipTests

# 运行（使用dev环境）
java -jar target/cateen-backend-1.0.0.jar --spring.profiles.active=dev
```

## 🔄 切换环境的方法

### 方法1：修改配置文件（推荐）

修改 `application.yml` 中的：
```yaml
spring:
  profiles:
    active: dev  # 改成 prod 就切换到生产环境
```

### 方法2：IDEA运行配置

1. 点击IDEA右上角的运行配置
2. 选择 `Edit Configurations...`
3. 在 `Environment` → `VM options` 中添加：
   ```
   -Dspring.profiles.active=dev
   ```

### 方法3：命令行参数

```bash
# 使用dev环境
java -jar app.jar --spring.profiles.active=dev

# 使用prod环境
java -jar app.jar --spring.profiles.active=prod
```

### 方法4：环境变量

```bash
# Windows PowerShell
$env:SPRING_PROFILES_ACTIVE="dev"
mvn spring-boot:run

# Mac/Linux
export SPRING_PROFILES_ACTIVE=dev
mvn spring-boot:run
```

## 🛠️ 配置文件的加载优先级

Spring Boot配置加载顺序：
```
application.yml（基础配置）
    ↓
application-{profile}.yml（覆盖/补充）
    ↓
命令行参数（最高优先级）
```

**示例：**
- `application.yml` 中配置 `server.port: 8080`
- `application-dev.yml` 中配置 `server.port: 8081`
- 最终启动端口是 **8081**（dev配置覆盖了主配置）

## 📋 常见问题

### Q1: 项目启动失败，提示数据库连接错误

**解决：**
1. 检查MySQL是否启动
2. 检查 `application-dev.yml` 中的数据库密码
3. 检查数据库名称是否为 `cateen`
4. 尝试手动连接：
   ```bash
   mysql -u root -p
   show databases;
   ```

### Q2: 修改了配置文件，但不生效

**解决：**
1. 重启项目（配置文件不支持热更新）
2. 确认修改的是正确的配置文件（dev还是prod）
3. 清理缓存：`mvn clean`

### Q3: 如何确认用的是哪个环境？

**方法1：** 查看启动日志
```
The following 1 profile is active: "dev"
```

**方法2：** 添加测试接口
```java
@RestController
public class TestController {
    @Value("${spring.profiles.active}")
    private String profile;
    
    @GetMapping("/env")
    public String getEnv() {
        return "当前环境：" + profile;
    }
}
```
访问：http://localhost:8080/api/env

### Q4: 想要添加自己的配置怎么办？

在对应的配置文件中添加即可：

**application-dev.yml:**
```yaml
# 自定义配置
my:
  config:
    upload-path: D:/upload/
    max-file-size: 10MB
```

**Java代码中使用：**
```java
@Value("${my.config.upload-path}")
private String uploadPath;
```

## 🎯 本地测试配置检查清单

在启动项目前，请确认：

- [ ] MySQL已启动
- [ ] 数据库 `cateen` 已创建
- [ ] 已执行 `schema.sql`（建表）
- [ ] 已执行 `data.sql`（初始化数据）
- [ ] `application.yml` 中 `active: dev`
- [ ] `application-dev.yml` 中数据库密码正确
- [ ] 端口8080没有被占用

**快速验证SQL：**
```sql
-- 检查数据库
SHOW DATABASES LIKE 'cateen';

-- 检查表
USE cateen;
SHOW TABLES;

-- 检查测试数据
SELECT * FROM profiles;
```

## 🚀 生产环境部署

部署到服务器时：

1. 修改 `application-prod.yml` 中的配置
2. 打包项目：`mvn clean package -DskipTests`
3. 上传 `target/cateen-backend-1.0.0.jar` 到服务器
4. 启动：
   ```bash
   java -jar cateen-backend-1.0.0.jar --spring.profiles.active=prod
   ```

或者在 `application.yml` 中直接改成：
```yaml
spring:
  profiles:
    active: prod
```
