# 数据统计功能完善说明

## 📋 问题总结

1. **评论数量不对等**：`foods` 表的 `rating_count` 和 `rating` 字段没有根据 `comments` 表的实际评论数量动态更新
2. **统计数据不真实**：浏览量、设备分布、来源统计等数据都是前端固定数据，没有从数据库查询

## ✅ 已完成的修改

### 1. 评论数量同步功能

#### 修改文件：
- `CommentServiceImpl.java`

#### 功能说明：
- 添加了 `updateFoodRatingStats()` 私有方法，用于更新美食的评分统计
- 在以下场景自动更新美食评分：
  - 创建评论时（如果自动通过）
  - 更新评论状态时（通过/拒绝状态变更）
  - 批量更新评论状态时
  - 删除评论时

#### 同步逻辑：
1. 统计该美食所有状态为 `approved` 的评论
2. 更新 `rating_count` 为评论总数
3. 计算平均评分并更新 `rating` 字段（保留1位小数）

## 🔄 自动更新机制

### 评论评分自动更新
评论的评分统计会在以下情况下自动更新：

1. **创建评论**：如果评论自动通过，立即更新美食评分
2. **更新评论状态**：从"待审核"改为"已通过"时更新评分
3. **批量更新状态**：批量审核通过评论时更新所有相关美食的评分
4. **删除评论**：删除评论后重新计算剩余评论的评分

### 统计数据更新
访问统计数据需要通过后端定时任务或日志分析系统定期写入数据库：

```sql
-- 示例：插入今天的访问统计
INSERT INTO visit_stats (id, stat_date, visits, unique_visitors, page_views, bounce_rate)
VALUES (
    CONCAT('vs_', DATE_FORMAT(CURDATE(), '%Y%m%d')),
    CURDATE(),
    1250,  -- 实际访问量
    480,   -- 实际独立访客数
    3200,  -- 实际页面浏览量
    25     -- 跳出率
) ON DUPLICATE KEY UPDATE 
    visits = VALUES(visits),
    unique_visitors = VALUES(unique_visitors),
    page_views = VALUES(page_views),
    bounce_rate = VALUES(bounce_rate);
```

## 📝 注意事项

1. **数据库兼容性**：SQL脚本使用MySQL语法，如果使用其他数据库需要调整
2. **初始数据**：提供的SQL脚本包含近30天的模拟数据用于演示
3. **生产环境**：建议配置定时任务定期采集真实的访问数据写入统计表
4. **性能优化**：如果数据量大，建议对统计表的 `stat_date` 字段建立索引
5. **数据迁移**：现有系统升级时必须先执行同步评分脚本
