# 微信餐厅点评小程序项目文档

## 📋 项目概述

### 项目名称
**餐厅点评小程序** (Restaurant Review Mini Program)

### 项目简介
一款集菜品展示、点评、收藏、智能推荐于一体的微信小程序，为用户提供便捷的餐厅菜品查询和评价服务，具有创新的转盘抽选功能和AI智能助手。

### 目标用户
- 美食爱好者
- 寻找餐厅推荐的用户
- 喜欢尝试新菜品的消费者
- 有选择困难症的用户（转盘功能）

---

## 🏗️ 功能架构

### 核心功能模块

#### 1. 首页模块
**功能描述**: 菜品展示和分类搜索
- **菜品展示**: 以卡片形式展示菜品信息
- **分类筛选**: 米饭类、面食类、快餐类等多种分类
- **搜索功能**: 支持菜品名称搜索
- **卡片信息**: 菜品照片、名称、评分、价格、所属店铺

#### 2. 菜品详情模块
**功能描述**: 菜品详细信息展示和互动
- **基本信息**: 菜品名、价格、所属店铺
- **收藏功能**: 星星收藏按钮，点击加入收藏
- **评论系统**: 用户评论展示
- **评分功能**: 五星评分系统（支持半星）

#### 3. 转盘抽选模块
**功能描述**: 个性化菜品抽选工具
- **自定义转盘**: 从"我的收藏"添加选项
- **转盘抽取**: 随机抽取菜品
- **结果展示**: 被抽中菜品卡片放大展示
- **重新抽取**: 支持多次抽取

#### 4. AI智能助手模块
**功能描述**: 智能菜品咨询服务
- **悬浮球入口**: 可拖拽的悬浮球交互
- **智能问答**: 基于数据库的菜品咨询
- **上下文理解**: 支持连续对话
- **推荐算法**: 基于用户偏好推荐

#### 5. 个人中心模块
**功能描述**: 用户个人信息和收藏管理
- **我的收藏**: 查看和管理收藏的菜品
- **个人信息**: 自定义网名和头像
- **隐私设置**: 用户权限管理
- **历史记录**: 浏览和评价历史

## 🗄️ 数据库设计

### 数据表结构

#### 菜品表 (dishes)
```sql
CREATE TABLE dishes (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '菜品名称',
    price DECIMAL(10,2) NOT NULL COMMENT '价格',
    image VARCHAR(255) COMMENT '菜品图片URL',
    description TEXT COMMENT '菜品描述',
    category_id INT COMMENT '分类ID',
    shop_id INT COMMENT '店铺ID',
    average_rating DECIMAL(3,1) DEFAULT 0 COMMENT '平均评分',
    total_ratings INT DEFAULT 0 COMMENT '总评价数',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### 分类表 (categories)
```sql
CREATE TABLE categories (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL COMMENT '分类名称',
    icon VARCHAR(255) COMMENT '分类图标',
    sort_order INT DEFAULT 0 COMMENT '排序',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 店铺表 (shops)
```sql
CREATE TABLE shops (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '店铺名称',
    address VARCHAR(255) COMMENT '地址',
    phone VARCHAR(20) COMMENT '电话',
    description TEXT COMMENT '店铺描述',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 用户表 (users)
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    openid VARCHAR(100) UNIQUE NOT NULL COMMENT '微信openid',
    nickname VARCHAR(100) COMMENT '昵称',
    avatar VARCHAR(255) COMMENT '头像URL',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### 收藏表 (favorites)
```sql
CREATE TABLE favorites (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL COMMENT '用户ID',
    dish_id INT NOT NULL COMMENT '菜品ID',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY unique_favorite (user_id, dish_id)
);
```

#### 评论表 (comments)
```sql
CREATE TABLE comments (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL COMMENT '用户ID',
    dish_id INT NOT NULL COMMENT '菜品ID',
    content TEXT COMMENT '评论内容',
    rating DECIMAL(2,1) NOT NULL COMMENT '评分(0.5-5.0)',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 📱 用户体验设计

### 交互流程

#### 首页浏览流程
1. **进入首页** → 展示推荐菜品
2. **分类筛选** → 点击分类标签筛选
3. **搜索功能** → 输入关键词搜索
4. **查看详情** → 点击卡片进入详情页

#### 收藏管理流程
1. **菜品收藏** → 详情页点击星星收藏
2. **转盘设置** → 从收藏中选择转盘选项
3. **抽取菜品** → 转盘随机抽取
4. **结果查看** → 展示抽中的菜品

#### 评价互动流程
1. **查看评价** → 详情页滚动到评论区
2. **发表评价** → 选择星级+输入评论
3. **提交评价** → 评价显示在列表中
4. **评分统计** → 实时更新平均分

### 性能优化

#### 图片优化
- **懒加载**: 进入可视区域才加载图片
- **压缩处理**: 自动压缩大尺寸图片
- **缓存策略**: 本地缓存常用图片
- **占位符**: 加载时显示占位图

#### 数据优化
- **分页加载**: 列表数据分页获取
- **缓存机制**: 缓存常用数据
- **预加载**: 预加载下一页数据
- **请求优化**: 防抖、节流处理

## 🔐 安全与权限

### 用户权限管理
- **基础信息**: 微信授权获取昵称、头像
- **可选权限**: 位置信息（推荐附近店铺）
- **相册权限**: 上传自定义头像
- **隐私保护**: 用户可控制信息展示

### 数据安全
- **接口鉴权**: JWT Token验证
- **数据加密**: 敏感信息加密存储
- **输入验证**: 防止SQL注入和XSS攻击
- **内容审核**: 评论内容自动审核

### 防作弊机制
- **评分限制**: 每用户每菜品只能评价一次
- **IP限制**: 防止恶意刷分
- **时间间隔**: 限制评价频率
- **举报功能**: 用户可举报不当内容
