# 开发环境配置说明

## 涉及的配置文件

| 文件 | 作用 |
|------|------|
| `cateen-backend/src/main/resources/application-dev.yml` | 后端上传路径和访问地址 |
| `cateen-mini/utils/config.js` | 小程序 API 请求地址 |

---

## 场景一：仅用浏览器/管理端调试（不涉及小程序）

后端和管理端都在同一台电脑上运行，`localhost` 可以互通。

**application-dev.yml**
```yaml
upload:
  path: D:/xawl_total/xawl_cateen/uploads
  base-url: http://localhost:8080
```

**config.js**（不需要改，默认 local 即可）
```js
function getMode() {
  var mode = wx.getStorageSync('API_ENV') || 'local';
  ...
}
```

---

## 场景二：小程序真机/模拟器调试（最常用）

小程序运行在手机或微信开发者工具模拟器上，无法访问 `localhost`，必须用电脑的局域网 IP。

**第一步：查询本机局域网 IP**

在电脑上运行：
```powershell
ipconfig | Select-String "IPv4"
```

选择和手机在同一个 WiFi 下的那个 IP，通常形如 `192.168.x.x`。  
本项目当前使用：`192.168.10.4`

**application-dev.yml**
```yaml
upload:
  path: D:/xawl_total/xawl_cateen/uploads
  base-url: http://192.168.10.4:8080   # 改为你的局域网 IP
```

**config.js**
```js
var ENVIRONMENTS = {
  local: 'http://localhost:8080/api/mini',
  lan:   'http://192.168.10.4:8080/api/mini',  # 改为你的局域网 IP
  prod:  'https://your-domain.com/api/mini'
};

function getMode() {
  var mode = wx.getStorageSync('API_ENV') || 'lan';  // 默认 lan
  ...
}
```

> 注意：修改 `base-url` 后需要重启后端。之前上传的图片 URL 已存入数据库（含旧 IP），需重新上传才能生效。

---

## 场景三：换了网络环境（IP 变了）

每次换 WiFi 或网络环境，局域网 IP 可能变化，需要同步修改两处：

1. `application-dev.yml` → `base-url`
2. `config.js` → `lan` 对应的地址

改完后重启后端即可。

---

## 场景四：生产部署

将后端部署到服务器后，改为服务器的公网域名/IP。

**application-dev.yml**（或新建 application-prod.yml）
```yaml
upload:
  path: /var/www/cateen/uploads   # Linux 服务器路径
  base-url: https://your-domain.com
```

**config.js**
```js
function getMode() {
  var mode = wx.getStorageSync('API_ENV') || 'prod';
  ...
}
```

---

## 快速排查图片无法显示

1. 在浏览器直接访问图片 URL，看是否能打开
2. 如果 URL 含 `localhost`，说明上传时 `base-url` 配置错误，改为局域网 IP 后重新上传
3. 如果 URL 含局域网 IP 但仍无法访问，检查防火墙是否放行了 8080 端口
4. 微信开发者工具需勾选「不校验合法域名」，否则非 https 域名会被拦截
